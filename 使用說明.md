# Jun 幣圈活動整理 — 使用說明 🌸

> 給 Jun 的白話操作手冊。不用懂程式，照著做就好。

---

## 1. 這個小App是什麼？

一個粉白風格的單頁網站，把你整理的幣圈活動做成漂亮卡片，
粉絲點連結進來就能看到活動說明、Jun 的心得、直接「複製邀請碼」、點按鈕去平台。

**檔案分工（5 個檔，放在同一個資料夾）：**

| 檔案 | 作用 | 你要不要動它 |
|---|---|---|
| `index.html` | 粉絲看到的頁面 | ❌ 不用動 |
| `admin.html` | **你的後台**，填表管理活動 | ✅ 你都在這裡操作 |
| `data.js` | 活動資料（後台會幫你產生） | ⚠️ 後台下載後拿來蓋掉它 |
| `app.js`、`styles.css` | 程式與美術 | ❌ 不用動 |

---

## 2. 先在自己電腦上看看（預覽）

直接用瀏覽器打開 `admin.html`（對著它按右鍵 → 開啟檔案 → Chrome）。
右邊會即時顯示粉絲看到的樣子。

> 小提醒：用「打開檔案」方式預覽，外觀都正常。
> 上線後（GitHub Pages）也是一樣的畫面。

---

## 3. 怎麼新增 / 修改活動（重點）

1. 打開 `admin.html`。
2. 左邊「➕ 新增活動」填表：
   - **平台**：選 Bitget / MEXC / Bybit…（決定卡片顏色）
   - **狀態**：進行中(紅) / 長期(綠) / 即將開始(橘) / 已結束(灰)
   - **活動標題、說明**：照你想講的寫
   - **數據列**：像「APR 72%」這種亮點，可加可不加
   - **Jun 說**：你的個人心得（留空就不顯示）
   - **活動連結、按鈕文字**
   - **邀請碼**：貼上你的真碼（沒有就留空）
3. 按 **「✓ 儲存這個活動」** → 右邊預覽立刻更新，下面清單也會出現。
4. 要改舊的：在「目前的活動清單」按 ✏️ 編輯；用 ↑ ↓ 調順序；🗑 刪除。

> 你的編輯會自動暫存在瀏覽器裡，關掉再打開還在。
> 但**這只是草稿**，要讓粉絲看到，必須做第 4 步上線。

---

## 4. 怎麼讓粉絲看到更新（上線）

1. 在後台按 **「💾 下載 data.js」** → 會下載一個 `data.js`。
2. 把這個新的 `data.js` **蓋掉**資料夾裡舊的 `data.js`。
3. 上傳到 GitHub（跟你更新幸運星一樣的動作）。
4. 等 1～2 分鐘，網站就更新了。

> 換句話說：**平常在後台填表 → 下載 data.js → 上傳 GitHub**，這三步就是你的固定流程。

---

## 5. 第一次部署到 GitHub Pages（拿到分享連結）

只做一次：

1. 到 GitHub 新增一個 repo，例如 `jun-crypto-hub`（建議用你已有的 `s6432891-star` 帳號）。
2. 把這個資料夾裡的 5 個檔案上傳進去。
3. repo → Settings → Pages → Source 選 `main` 分支、根目錄 `/` → Save。
4. 約 1～2 分鐘後會得到網址，類似：
   ```
   https://s6432891-star.github.io/jun-crypto-hub/
   ```
5. 這個網址就是你分享給粉絲、放 Threads / IG / Portaly 的連結。

> 需要我幫你把這 5 個檔變成可直接 push 的 git 專案、或寫好上傳指令，再跟我說。

## 6. 常見問題

- **邀請碼還是 YOUR_CODE？** 那是假碼。在後台對應活動按 ✏️，把邀請碼和連結換成真的，再下載上傳。
- **想改網站名字 / 標語 / Threads 連結？** 後台最上面「⚙️ 網站基本設定」。
- **不小心改壞了想還原？** 後台按「↺ 重新載入線上版」，會回到目前 `data.js` 的內容。
- **風險提醒一定要留著**：免責聲明那行是你的品牌原則，別拿掉。
